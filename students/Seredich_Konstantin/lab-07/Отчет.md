## Цель работы

Реализовать **CQRS** с разделением **Write Model** и **Read Model** и синхронизацией на событиях.

---

## Вариант №28 — «Говорю красиво» 🗣️

Система карточных колод: запись состояния агрегата и быстрые списки для UI.

## Ход выполнения работы

### 1. Write Model

**Агрегат:** `Deck` — нормализованное хранение, инварианты при изменении.

Ответственность: консистентность и правила предметной области при командах.

---

### 2. Read Model

**Проекция / представление:** таблица **`deck_summaries`** (или эквивалент в проекте).

- Денормализованное поле **`cards_count`** для экранов списков без тяжёлых `COUNT`/`JOIN` при каждом чтении.

**Read-доступ:** [src/cqrs/read_model/deck_view.py](src/cqrs/read_model/deck_view.py).

---

### 3. Event-Driven Sync

При сохранении карточки генерируется **`CardAddedToDeckEvent`**. Обработчик проекции **`DeckProjectionHandler`** увеличивает счётчик в read-модели.

Код проекции: [src/cqrs/projection/deck_projection.py](src/cqrs/projection/deck_projection.py).

Обеспечивается **eventual consistency**: список кратковременно может отставать от записи, но сходится при отсутствии новых событий.

---

### 4. Оптимизация запросов

- Индекс **B-Tree** по **`owner_id`** в таблице чтения — см. [src/cqrs/infrastructure/sql/optimization.sql](src/cqrs/infrastructure/sql/optimization.sql).
- **`EXPLAIN ANALYZE`:** ожидается **Index Scan** вместо последовательного сканирования при типовых запросах списка.

---

### 5. Тесты проекций

[tests/test_projection.py](tests/test_projection.py) — проверка обновления read-модели при событии.

## Контрольные вопросы

1. **CQRS vs CQS:** CQS — разделение методов на команды и запросы в коде; CQRS — разделение моделей/путей чтения и записи на архитектурном уровне.

2. **Почему Read Model денормализована:** ускорение типовых чтений; цена — усложнение пути записи/проекции.

3. **Синхронизация при сбое:** outbox / брокер с повторной доставкой; восстановление проекции из журнала событий.

4. **Eventual consistency:** при остановке обновлений состояния через конечное время согласуются.
